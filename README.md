# Todo App на React

## Описание проекта

Todo App - это современное веб-приложение для управления задачами, разработанное с использованием React. Приложение позволяет пользователям создавать, редактировать, удалять и фильтровать задачи, а также отслеживать статистику выполненных и активных задач.

## Функциональные возможности

### Основной функционал
- **Добавление задач**: Пользователи могут добавлять новые задачи с помощью формы ввода
- **Редактирование задач**: Возможность редактировать текст задачи после её создания
- **Удаление задач**: Удаление отдельных задач или очистка выполненных задач
- **Переключение статуса**: Отметка задач как выполненных/невыполненных с помощью чекбоксов
- **Фильтрация задач**: Просмотр всех, активных или выполненных задач
- **Статистика**: Отображение количества всех, активных и выполненных задач

### Дополнительные возможности
- **Горячие клавиши**: 
  - `Enter` - добавление задачи или сохранение редактируемой задачи
  - `Esc` - отмена редактирования задачи
- **Адаптивный дизайн**: Приложение корректно отображается на различных устройствах
- **Визуальная обратная связь**: Анимации и изменения стилей при взаимодействии с элементами

### Компоненты приложения

#### 1. App.jsx
Основной компонент приложения, который управляет глобальным состоянием:
- Хранит массив задач (`todos`)
- Управляет фильтрацией задач (`filter`)
- Содержит основные функции:
  - `addTodo()` - добавление новой задачи
  - `deleteItem()` - удаление задачи
  - `toggleItem()` - переключение статуса задачи
  - `editTodo()` - редактирование текста задачи
  - `clearCompleted()` - очистка выполненных задач

#### 2. TodoInput.jsx
Компонент для ввода новых задач:
- Текстовое поле для ввода задачи
- Кнопка "Добавить"
- Поддержка добавления задач по нажатию Enter
- Проверка на пустые значения

#### 3. TodoList.jsx
Компонент, отвечающий за отображение списка задач:
- Отображает задачи в виде списка
- Передает пропсы дочерним компонентам TodoItem

#### 4. TodoItem.jsx
Компонент отдельной задачи:
- Отображает текст задачи
- Чекбокс для переключения статуса
- Кнопки "Редактировать" и "Удалить"
- Режим редактирования с возможностью изменения текста
- Поддержка клавиш Enter (сохранение) и Esc (отмена)

#### 5. TodoFilters.jsx
Компонент для фильтрации задач:
- Кнопки для переключения между фильтрами:
  - "Все" - отображение всех задач
  - "Активные" - только невыполненные задачи
  - "Выполненные" - только выполненные задачи

#### 6. TodoStats.jsx
Компонент статистики:
- Отображает количество всех, активных и выполненных задач
- Кнопка "Очистить выполненные" для удаления всех выполненных задач

### Требования

```bash

```

## Архитектурные решения

### Управление состоянием
- Используется React хук `useState` для управления локальным состоянием
- Центральное состояние задач находится в компоненте `App`
- Функции для изменения состояния передаются в дочерние компоненты через пропсы

### Фильтрация задач
- Фильтрация происходит на уровне компонента `App`
- Массив задач фильтруется в зависимости от выбранного фильтра
- Отфильтрованный массив передается в `TodoList`

### Обработка событий
- Все обработчики событий определены в соответствующих компонентах
- Используются стрелочные функции для правильного контекста `this`
- Обработка клавиш (Enter, Esc) для улучшения UX

## Стилизация

### CSS-файлы
- `App.css` - основные стили приложения
- `index.css` - глобальные стили (если есть)

### Особенности стилизации
- Используется Flexbox для создания адаптивных макетов
- Внедрены CSS-анимации и переходы для улучшения UX
- Адаптивный дизайн для мобильных устройств
- Использование теней и скруглений для улучшения визуального восприятия

## Структура данных

### Объект задачи
Каждая задача представлена объектом со следующими свойствами:
```javascript
{
  id: number,        // Уникальный идентификатор (создается с помощью Date.now())
  text: string,      // Текст задачи
  completed: boolean // Статус выполнения (true/false)
}
```

## Принципы разработки

### Функциональные компоненты
- Все компоненты реализованы как функциональные компоненты
- Используются React хуки для управления состоянием и побочными эффектами

### Декомпозиция
- Приложение разделено на небольшие, переиспользуемые компоненты
- Каждый компонент отвечает за определенную функциональность

### Управление пропсами
- Данные передаются сверху вниз через пропсы
- Функции обратного вызова передаются для изменения состояния родительского компонента
